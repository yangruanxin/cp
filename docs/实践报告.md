## 2. 总体架构

本项目实现 ToyC 到 RISC-V32 汇编的编译器。编译器从标准输入读取 ToyC 源码，经过词法分析、语法分析、语义分析、IR 生成、可选 IR 优化和代码生成，最终将汇编输出到标准输出。

```text
ToyC source
    -> Lexer
    -> Parser
    -> AST
    -> Semantic + IRGenerator
    -> Optimizer (-opt)
    -> CodeGenerator
    -> RISC-V32 assembly
```

项目采用线性 IR 作为模块边界。A 只负责源码到 AST，B 只负责 AST 到 IR，C 只负责 IR 到汇编，D 负责把完整链路串起来，并在 `-opt` 参数开启时对 IR 做优化。

## 3. 各模块实现

### 3.1 词法分析

词法分析器识别 ToyC 的关键字、标识符、整数字面量、运算符、关系运算符、逻辑运算符、括号、花括号、分号、逗号和文件结束标记。空白字符会被跳过，单行注释和多行注释不会进入 token 序列。非法字符和未闭合注释会抛出异常并携带行列信息。

### 3.2 语法分析

语法分析器采用递归下降实现，按照 ToyC 文法构造 AST。表达式解析按优先级分层处理，支持逻辑或、逻辑与、关系、加减、乘除模、一元运算、括号表达式和函数调用。语句支持块语句、空语句、表达式语句、赋值、变量/常量声明、`if/else`、`while`、`break`、`continue` 和 `return`。

### 3.3 语义分析与 IR 生成

语义分析阶段维护嵌套符号表，检查变量和函数重定义、未定义标识符、常量赋值、函数参数数量、`break/continue` 位置和返回路径等规则。IR 生成阶段将 AST 降低为三地址形式的线性 IR，控制流统一表示为 `LABEL`、`GOTO` 和 `IF_GOTO`，函数调用通过 `ARG` 和 `CALL` 表示。

### 3.4 代码生成

代码生成模块将线性 IR 翻译为 RISC-V32 汇编。当前实现采用正确性优先的栈槽策略：每个临时量、局部变量和形参都分配独立栈槽，指令执行时从栈或全局内存载入 scratch 寄存器，计算后立即写回。函数入口生成栈帧和返回地址保存，函数出口统一通过 epilogue 恢复栈帧并返回。

### 3.5 优化

优化器在 `-opt` 参数开启时执行，当前包含两个 IR 级优化 pass：

| 优化 | 说明 |
|------|------|
| 常量折叠 | 对 `ADD/SUB/MUL/DIV/MOD`、关系运算、逻辑运算和一元 `NEG/NOT` 在操作数均为常量时直接计算结果 |
| 死代码删除 | 删除结果从未被使用且没有副作用的临时量定义，并迭代执行直到 IR 长度稳定 |

为保证正确性，优化器在遇到函数边界、标签、跳转、条件跳转和函数调用时清空常量传播状态，避免把某条控制流路径上的常量误传播到其他路径。`STORE`、`CALL`、`RETURN`、控制流指令、`DECL` 和 `ARG` 会被视为有副作用或结构性指令，不会因为目标临时量未使用而删除。

## 4. 遇到的问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 集成测试链接失败，提示重复定义 `main` | 将 CMake 中的核心源码拆成不含 `src/main.cpp` 的 `COMPILER_LIB_SOURCES`，主程序和集成测试分别链接该源码集合 |
| 常量传播可能跨过 label 或 branch 造成错误优化 | 遇到控制流边界时清空常量表，只做局部基本块内可证明安全的折叠 |
| 死代码删除可能误删函数调用或参数传递 | 明确副作用指令集合，保留 `CALL`、`ARG`、`STORE`、`RETURN` 和控制流相关 IR |
| 一遍死代码删除无法完全删除链式无用临时量 | 在 `optimize()` 中循环执行 DCE，直到指令数量不再变化 |

### 5.2 性能测试

当前优化器已经接入 `-opt` 参数并能减少部分常量表达式和无用临时量产生的汇编。性能测试仍需在课程评测环境或 RISC-V 运行环境中与 gcc -O2 基线进行对比，报告最终版应补充实际运行时间数据。

## 6. 总结与反思

本项目通过 AST 和线性 IR 将四个模块解耦，使前端、语义、后端和优化可以并行开发。D 侧重点是保证工程能稳定构建、主入口能串联完整链路、优化器在正确性优先的前提下改善 IR，以及通过集成测试及时发现模块契约不一致。后续如果继续提升性能，可以在现有 IR 上增加基本块划分、公共子表达式消除、更精细的活跃变量分析和寄存器分配优化。
